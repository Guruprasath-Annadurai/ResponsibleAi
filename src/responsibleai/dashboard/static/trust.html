🔐 Security posture at a glance

Authentication & Access

OIDC SSO with enforceable SSO-only mode. 4-tier RBAC (Owner/Admin/Analyst/Viewer) on every endpoint. API keys stored as SHA-256 hashes only, never in plaintext.

● Implemented

Multi-tenant isolation

Every governance data table carries an org_id and every query filters by it. Cross-org leakage is treated as a security defect.

● Implemented

Audit trail integrity

Hash-chained audit log detects direct database tampering. Exportable for SIEM. Does not defend against a fully compromised database — stated honestly, not oversold.

● Implemented

Encryption at rest

Deployer's responsibility (disk/volume encryption) — not enforced by the application layer. No field-level encryption.

● Partial — infra-dependent

Dependency scanning

pip-audit runs on every CI build against the full resolved dependency tree. Findings are triaged and documented, not silently ignored.

● Implemented

Third-party penetration test

Not yet performed — cost-gated. An automated OWASP ZAP baseline scan is available as an interim measure and explicitly disclosed as such, not a substitute.

● Not started
